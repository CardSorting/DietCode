# Security Policy: Vulnerability Disclosure 🛡️

The integrity of the DietCode Hive is our highest priority. We appreciate all reports of security vulnerabilities and handle them with Level 10 rigor.

---

## 1. Supported Versions

We only provide security updates for the current version.

| Version | Status |
| :--- | :--- |
| **V1.0.0** | ✅ Supported (Hive Optimized) |
| < 1.0.0 | ❌ End of Life |

---

## 2. Reporting a Vulnerability

If you discover a vulnerability that compromises the Sovereign architecture or source insulation, please report it via one of the following methods:

- **Private Issue**: Use the "Private Vulnerability Reporting" feature in GitHub if available.
- **Direct Communication**: Contact the repository maintainers directly.

**Do not open a public issue for security-sensitive vulnerabilities.** We will work with you to remediate the issue privately before a public disclosure is made.

---

## 3. Remediating Integrity Regressions

Vulnerabilities will be addressed in a dedicated **Integrity Patch**. These patches are prioritized above all feature development.

- **Fast Track**: Critical data corruption or source-leakage vulnerabilities are patched within 48 hours.
- **Moderate Track**: Information disclosure or performance-sink vulnerabilities are patched within 7 days.

---
**Thank you for protecting the Sovereignty of the Hive.**
